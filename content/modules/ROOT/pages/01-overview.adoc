== The Customer Problem

Software supply chain attacks are the fastest-growing category of cyber threat.
High-profile incidents like SolarWinds and Log4Shell showed that attackers no longer break in -- they sneak in through dependencies and build pipelines.

Customers face three compounding pressures:

* *Regulatory pressure* -- Executive Order 14028, NIST SSDF, and emerging EU requirements mandate SBOMs and provenance for software shipped to government and regulated industries.
* *Audit burden* -- Security teams cannot verify what is in a container image, when it was built, or whether it was tampered with.
* *Developer friction* -- Bolting signing and scanning onto existing pipelines is painful, fragile, and inconsistent.

The result: teams either skip signing entirely, or spend months building custom tooling that breaks with every upstream change.

== The {TSFName} Story

{TSFShortName} eliminates all three problems with a single integrated platform:

* *Konflux* orchestrates every build, test, and release workflow
* *{RHTASName} (RHTAS)* signs every artifact cryptographically with short-lived certificates -- no long-lived keys to rotate or compromise
* *{RHTPAName} (RHTPA)* generates a full Software Bill of Materials (SBOM) and scans for vulnerabilities automatically
* Every image gets SLSA (Supply-chain Levels for Software Artifacts) Level 3 provenance -- an immutable, signed record of who built it, when, from what source, and using which pipeline

The presenter value proposition in one sentence: *"This is the same platform Red Hat uses to build its products -- from RHEL to {OCPName} -- now you can use it too."*

== Common Customer Questions

*"How is this different from just running Tekton pipelines?"*
→ Tekton is the engine. {TSFShortName} adds the signing infrastructure (Fulcio/Rekor/TUF), SBOM generation, vulnerability scanning, release management, and a unified UI -- all wired together out of the box.

*"Does Red Hat actually use this internally?"*
→ Yes. Konflux is the platform Red Hat uses to build most of its software -- from {OCPName} and RHEL to Ansible. {TSFShortName} is that platform packaged for customer deployment.

*"What is SLSA Level 3 and why does it matter?"*
→ SLSA Level 3 is the highest provenance standard that can be achieved without hardware roots of trust. It proves the build ran in an isolated, ephemeral environment using a specific pipeline definition -- making it impossible to forge without compromising the build system itself.

*"We already have a signing process. Why replace it?"*
→ Most existing signing processes use long-lived keys that must be rotated, stored securely, and audited. {TSFShortName} uses keyless signing via Sigstore -- short-lived certificates tied to your OIDC identity. No keys to manage, no keys to compromise.

*"What happens when a new CVE drops?"*
→ RHTPA continuously scans SBOMs against vulnerability databases. When a new CVE is published, you immediately see which images are affected -- without re-scanning from scratch.
