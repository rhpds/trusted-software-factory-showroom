= How to Use This Demo Guide

This guide helps sales engineers and technical presenters deliver an effective {TSFName} ({TSFShortName}) demonstration to customers.

Every demo is different -- adapt this guide to your specific audience and time constraints.
If you have less time than noted, you can skip or shorten individual sections.

== Demo Overview

The *Demo Overview* section provides the business scenario, background, and common customer questions to be ready for.

[IMPORTANT]
====
Read this section thoroughly before presenting. This demo focuses on business value -- the *why* behind secure software supply chains -- not just technical features.
====

== Demo Details

The *Demo Details* section is your single reference for:

* Proposed timing
* Products and versions
* Environment access and credentials
* Troubleshooting tips
* Follow-up resources
* Author contacts

== Demo Modules

This is the main demo content. Each section follows a *Know* and *Show* structure:

=== Know

The *Know* section gives you business context and background. It is not a script to read aloud -- understand it so you can speak naturally to your customer's situation.

You'll typically find:

* Business challenge
* Technical solution
* Business value and outcomes
* Common objections

=== Show

The *Show* section is step-by-step. Follow these presenter actions in order during the live demonstration.

== Optional: Build Your Own Adventure

Appendices A–C cover the full {TSFShortName} installation process for presenters who want to demonstrate a live install on their own cluster.
The standard demo uses a pre-provisioned RHDP environment where {TSFShortName} is already deployed -- you do not need the appendix for the main demo flow.

== Troubleshooting

If you encounter any issues during the demonstration, review Appendix D for solutions. 

== Let's Get Started

Begin with the *Demo Overview*, then walk through the demo modules in order.
