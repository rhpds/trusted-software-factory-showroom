== Key Concepts Recap

=== Konflux

Konflux is the orchestration layer that ties the secure software supply chain together. It provides application and component management, pipeline definitions, release plans, and integration with source control and registries.

=== SLSA Provenance

Supply-chain Levels for Software Artifacts (SLSA) Level 3 provenance:

* *Non-falsifiable* -- signed by the build system, cannot be forged
* *Complete* -- records all inputs and build steps
* *Isolated* -- build runs in an ephemeral, hermetic environment

=== {RHTASName}

Red Hat Trusted Artifact Signer (RHTAS), based on Sigstore, provides:

* *Fulcio* -- certificate authority that issues short-lived signing certificates
* *Rekor* -- transparency log that records every signature permanently
* *TUF* -- trust root framework for secure distribution of signing metadata

Signatures are tied to build identity via OIDC -- no long-lived keys to manage or compromise.

=== {RHTPAName}

Red Hat Trusted Profile Analyzer (RHTPA) generates:

* *SBOMs* -- complete inventory of every software component
* *Vulnerability reports* -- CVEs affecting packages in every image
* *License information* -- for open-source compliance and legal review
