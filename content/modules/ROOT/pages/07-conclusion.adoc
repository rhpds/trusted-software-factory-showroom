== Key concepts recap

=== Konflux

Konflux is the orchestration layer that ties everything together. It provides:

* Application and component management
* Pipeline definitions and triggers
* Release plans and strategies
* Integration with source control and registries

=== SLSA provenance

SLSA (Supply-chain Levels for Software Artifacts) Level 3 provenance provides:

* **Non-falsifiable** -- Signed by build system, cannot be forged
* **Complete** -- Records all inputs and build steps
* **Isolated** -- Build runs in ephemeral environment

This enables you to verify *exactly* how an image was built, from which source, by which pipeline.

=== {RHTASName}

Red Hat Trusted Artifact Signer (based on Sigstore) provides:

* **Fulcio** -- Certificate authority that issues short-lived signing certificates
* **Rekor** -- Transparency log that records all signatures
* **TUF** -- Framework for secure distribution of trust roots

Signatures are tied to your identity (via OIDC), not long-lived keys that can be compromised.

=== {RHTPAName}

Red Hat Trusted Profile Analyzer generates:

* **SBOMs** -- Complete inventory of all software components
* **Vulnerability reports** -- Known CVEs affecting your images
* **License information** -- Compliance and legal review data

This enables proactive security management and rapid response to new vulnerabilities.

== Common troubleshooting scenarios

=== Issue: Quay token error during deployment

**Problem:** The `tsf deploy` command fails with "token field not found in secret"

**Root cause:** Expired, invalid, or incorrectly saved Quay OAuth token

**Solution:**

. Regenerate the Quay OAuth token in Quay UI
. Remove the secret:
+
[source,role="execute"]
----
oc delete secret tsf-quay-integration -n tsf
----

. Re-run Quay integration:
+
[source,role="execute"]
----
tsf integration quay \
  --organization="$QUAY__ORG" \
  --token="$QUAY__API_TOKEN" \
  --url="$QUAY__URL"
----

. Restart deployment:
+
[source,role="execute"]
----
tsf deploy
----

=== Issue: Cert-Manager subscription conflict

**Problem:** Deployment fails due to subscription conflict for Cert-Manager operator

**Root cause:** Red Hat Cert-Manager Operator already installed on cluster

**Solution:**

Edit the `tsf-config` ConfigMap and change Cert-Manager's `manageSubscription` from `true` to `false`:

[source,role="execute"]
----
oc edit configmap tsf-config -n tsf
----

Find the Cert-Manager component section and modify:
[source,yaml]
----
cert-manager:
  enabled: true
  manageSubscription: false  # Change this from true to false
----

Save and restart deployment.

=== Issue: RHTPA UI URL inaccessible

**Problem:** URL displays "server%s(<nil>)" or returns an error

**Root cause:** RHTPA service or PostgreSQL dependency failed to start

**Diagnostic commands:**

. Check RHTPA operator status:
+
[source,role="execute"]
----
oc get pods -n tsf-tpa
----

. Check PostgreSQL:
+
[source,role="execute"]
----
oc get pods -n tsf-tpa | grep postgres
----

. Review operator logs:
+
[source,role="execute"]
----
oc logs -n tsf-tpa deployment/rhtpa-operator
----

. Locate correct URL:
+
[source,role="execute"]
----
oc get routes -n tsf-tpa
----

=== Issue: Deployment appears to hang

**Problem:** Deploy command runs indefinitely without output

**Root cause:** Extended processing time for Helm chart deployment and container image pulls

**Resolution:**

This is normal behavior. Monitor progress separately:

[source,role="execute"]
----
watch oc get pods -A
----

Typical completion time is approximately 15 minutes. Large images (Keycloak, RHTPA) take time to pull.

=== Issue: Browser fails during GitHub integration

**Problem:** Command logs error about missing "xdg-open" executable

**Root cause:** Installer container lacks graphical browser capability

**Solution:**

Manually copy the `localhost:8228` URL from the terminal output and open it in your web browser. The port is already exposed by the `podman run -p 8228:8228` command.

=== Issue: Pipeline fails with "cannot push to Quay"

**Problem:** Build pipeline succeeds but push to Quay fails

**Root cause:** Quay organization doesn't exist or token lacks permissions

**Solution:**

. Verify the organization exists in Quay
. Verify the OAuth token has write permissions
. Check the Quay integration secret:
+
[source,role="execute"]
----
oc get secret tsf-quay-integration -n tsf -o yaml
----

. If needed, recreate the integration:
+
[source,role="execute"]
----
oc delete secret tsf-quay-integration -n tsf
tsf integration quay \
  --organization="$QUAY__ORG" \
  --token="$QUAY__API_TOKEN" \
  --url="$QUAY__URL"
----

=== Issue: Component not triggering builds

**Problem:** Merged PR but pipeline never runs

**Root cause:** Webhook not configured or not reaching cluster

**Solution:**

. Check webhook configuration in GitHub/GitLab
. Verify cluster is publicly accessible
. Check Tekton EventListener:
+
[source,role="execute"]
----
oc get eventlistener -A
----

. Check recent webhook deliveries in GitHub/GitLab settings
