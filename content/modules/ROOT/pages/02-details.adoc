== Technical requirements

=== Cluster requirements

To deploy {TSFShortName}, you need:

* **{OCPName} cluster:** Version {OCPVersion} or later
* **Cluster topology:** Standard 3-node or larger deployment (Single Node OpenShift not supported)
* **Cluster state:** Fresh cluster without existing production workloads
* **Access level:** Cluster-admin privileges required
* **Network connectivity:** Bidirectional access to external services (GitHub/GitLab, Quay)

=== External service accounts

You will need accounts and credentials for:

GitHub (recommended)::
* GitHub organization (github.com)
* Ability to create GitHub Apps in the organization
* Network connectivity for webhook delivery to cluster

GitLab (alternative)::
* GitLab instance (SaaS or self-hosted)
* Project Access Token with `api`, `read_repository`, `write_repository` scopes
* Maintainer role on the project

Quay registry::
* Quay account (quay.io or self-hosted Quay instance)
* Organization created in Quay
* OAuth access token with full permission scopes

=== Local workstation tools

Your local machine needs:

* **{OCPCli} CLI:** OpenShift command-line tool installed and configured
* **Podman:** Container runtime (Docker is not supported)
* **Web browser:** Modern browser for accessing Konflux UI
* **Text editor:** For creating and editing configuration files

== Workshop environment

=== Provided cluster

For this workshop, you have access to:

* **{OCPName} cluster:** {OCPVersion}
* **Console URL:** {openshift_console_url}
* **Cluster-admin access:** Full administrative privileges
* **Internet connectivity:** Access to GitHub, Quay, and other external services

=== Components deployed

During this workshop, you will deploy these components:

Cert-Manager Operator::
Certificate lifecycle management for TLS certificates

Red Hat Keycloak::
Identity and access management with SSO

Konflux Operator::
Custom Resource Definitions for build pipelines

Konflux UI::
Web interface for managing applications and builds

OpenShift Pipelines::
Tekton-based CI/CD pipelines

{RHTASName}::
Artifact signing with Fulcio, Rekor, and TUF

{RHTPAName}::
SBOM generation and vulnerability analysis

Quay integration::
Container registry credentials management

=== Namespaces created

The installation creates these namespaces:

* `tsf` -- Core {TSFShortName} configuration
* `cert-manager-operator` -- Certificate management
* `rhbk-operator` -- Keycloak operator
* `rhtpa-operator` -- Trusted Profile Analyzer operator
* `konflux-ui` -- Web interface
* `openshift-storage` -- Storage components
* Additional namespaces for Konflux components
